Skill requirements

Building a Linux system from the ground up is not a trivial task, and it requires familiarity with the Linux command line (shell). As a minimum, you should be able to do basic command line operations such as copying/moving/renaming files, changing/listing directories, printing file contents, deleting files, creating files/directories, etc. You are also expected to edit files from the command line, using a text editor such as GNU Nano or Vi/Vim.

If you've never worked with the command line or Linux in general, I suggest you to visit linux-tutorial.info to learn some basic knowledge first.

You also need to know the basic of software compilation (such as using gcc, make, etc.). Here's good website about the subject: www.tldp.org


Host requirements

You need to have a working Linux environment (Ideally a mainstream distribution such as Debian or Arch) setup on your machine. It can either be a bare-metal hard drive install or can be in a virtual machine. It needs to have at least 2GB of RAM and 15GB of free disk space in order to build all the packages.

Here's a list of extra packages you need to install on your system:

  • gcc (The GNU compiler collection)
  • make (a build tool that simplifies the compilation process for users
  • bc (needed for the building of the Linux kernel)
  • qemu (Quick Emulator, used to run and test MicroLinux
  • ncurses (a menu interface for configuration scripts)
  • gawk (a text-processing language)
